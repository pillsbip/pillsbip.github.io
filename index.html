Analysis of customer and employee data

I am currently working at iSelect-Analytics and Data Services Division. My project involves analysis of customer and employee data. I am unable to provide sample results as they are iSelect intellectual property. I have completed the following tasks:

  • I have developed a software package for predicting probability of lead-to-sales conversion, based on customer response to the questions on the revamped online comparison form for iSelect broadband products. By utilizing information related to postcodes and email domains, I was able to achieve the same predictive efficiency as the previous model, despite access to only 10% of data for the training set compared to the previous model.
  • I have developed a software package for skills-based routing (SBR) of customer calls to agents. This software package clusters agents based on statistical comparison of their conversion performance. In case of inbound calls, the algorithm needs to choose the right agent based on customer response to three or four questions. My algorithm has been shown to lead to an increase in lead-to-sales conversion by 20%. Both packages above have been implemented using R and T-SQL.
  • I have built a model for predicting time-dependent propensity of leads-to-sales conversion using Python, Tensor Flow and T-SQL. I was able to demonstrate an increase in predictive capacity compared to the model that uses time-independent propensity (auc increase from 0.63 to 0.65).
  • I have performed a statistical comparison of agent performance (using R, with respect to lead-to-sales conversion) as a result of two different training programs and has demonstrated that there is no significant difference in agent performance, based on a 95% confidence interval.
  • iSelect has recently moved to Salesforce customer resource management (CRM) which stores information in JSON format . I have written T-SQL code so as to interface with Salesforce CRM and extract information for reporting purposes. The reports are prepared as Tableau dashboards.

Machine learning for speech therapy

In this project, I investigate the use of machine learning for speech therapy. I am currently working on this project in collaboration with Dr. Ken Pang from Gender Service Division of Royal Children's Hospital. The initial application deals with assigning a score to a person's voice that rates its masculinity. This rating is generated by comparison with a data set of male and female voices. The comparison is based on acoustic parameters generated using signal processing. We have used variable reduction techniques to shortlist relevant acoustic parameters. Some of the files used for implementing this project are available on github. This project involves use of R for implementing machine learning. Node.JS and AngularJS are being used for building the web interface for accessing the voice files of the person undergoing speech therapy. The figure below provides the confusion matrix for a set of 950 male and female voices for two of the machine learning algorithms being used. The results show that the algorithms being used are able to provide predictions with high accuracy.

Energy consumption of data mining

I have created a blog (available ​ here ) on energy consumption of data mining algorithms. This blog was the result of the work that I carried out at Centre for Energy-Efficient Telecommunications (CEET). I investigated energy consumption of supervised (regression, forecasting etc) and unsupervised (clustering, correlation etc) machine learning algorithms and platforms for their implementation. This project involved use of R as well as SAS Studio for scripting the statistical analysis algorithms. SQL was used for database management, including accessing the data for processing. For example, I have shown that SAS University Edition consumes more energy than R for algorithm implementation (as shown in the illustrative figure below corresponding to box plot generation). This is due to the fact that SAS University Edition is implemented as a virtual machine, leading to energy overheads associated with virtualization.

Analysis of social media sentiment

This is a personal interest project I have worked on after participation in the kaggle competition (featuring text mining) organized by Data Science Melbourne in April 2016.I have built software in R (based on twitteR package and RTextTools package) that downloads tweets from Twitter containing a particular phrase and then evaluates the sentiment associated with that phrase. For example, the plot below corresponds to the sentiment on Twitter corresponding to the phrase 'negative gearing' over a three day period in July. The difference between the predicted and actual fraction of tweets against negative gearing was found to be less than 10%. Thus my software package predicts sentiment with reasonable accuracy.
